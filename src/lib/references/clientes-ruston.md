fullText contains '[NOME_CLIENTE]' and fullText contains 'FCA'
fullText contains '[NOME_CLIENTE]' and fullText contains 'call'
```

### Gerais (Time Lucas)
```
fullText contains 'FCA'                          — Últimas FCAs (modificação recente)
fullText contains 'Check-in'                     — Últimos check-ins com clientes
fullText contains 'call'                         — Calls recentes, insights
fullText contains 'Relatório'                    — Relatórios performance anterior
```

### O Que Extrair de Cada Tipo
- **FCA:** Fatos identificados, Causas raiz, Ações combinadas, ROI esperado
- **Check-in:** Status cliente, métricas atuais vs. baseline, problemas, prioridades
- **Calls:** Decisões tomadas, compromissos, feedback cliente, objeções
- **Relatórios:** Métricas históricas, tendências, benchmarks internos

## INTEGRAÇÃO GOOGLE DRIVE — PROTOCOLO COMPLETO

Ao receber qualquer solicitação de cliente:

1. **Busque contexto** (Check-in + FCA + Calls últimas 3)
   - `fullText contains 'Check-in' and fullText contains '[CLIENTE]'`
   - `fullText contains 'FCA' and fullText contains '[CLIENTE]'`
   - `fullText contains '[CLIENTE]' and fullText contains 'call'`

2. **Leia documentos** com `google_drive_fetch` (máx. 3 documentos)

3. **Extraia insights:** decisões, problemas, restrições TOC, ROI, métricas atuais

4. **Crie arquivo** `briefings/contexto-gdrive-YYYY-MM-DD.md` com resumo estruturado:
   ```
   ## 📡 CONTEXTO GOOGLE DRIVE — [CLIENTE]
   📅 Atualizado: [Data]

   - Fato: [...]
   - Causa: [...]

   - ROI: [...]
   
   ### Último Check-in ([Data])
   - Status: [Verde/Amarelo/Vermelho]
   - Métrica-chave: [Valor] (baseline: [valor])
   - Problema principal: [...]
   - Próximos passos: [...]
   
   ### Insights para Diagnóstico
   - Restrição: [...]
   - Oportunidade: [...]

   ### Fontes
